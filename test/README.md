# test/ — 대상 프로젝트 테스트 (Test)

> 이 저장소로 개발하는 **대상 프로젝트의 테스트** 자리.

## 목적
대상 프로젝트의 유닛/통합 테스트를 담는다. Tester가 `docs/sot/test-design/`의 **테스트 설계**(테스트 조건·커버리지 목표·스위트 구조·절차 원칙)를 구체 케이스·데이터·픽스처·실행 스크립트로 구현한다. 처음엔 비어 있다. (테스트 설계 = 무엇을 어떻게 검증할지의 SoT, `test/` = 그 구현.)

## 필수 내용
- `docs/sot/test-design/`의 테스트 조건(`TD-n`)·블랙박스 커버리지 목표를 실행 가능한 케이스로 구현한 테스트. `src/` 구현·`docs/sot/specification/`에 대응한다.
- 화이트박스 커버리지(결정/분기/MC-DC) 측정은 여기서 한다(설계 SoT의 대상이 아니다).
- 테스트 레이아웃·실행 방법은 대상 프로젝트 규약을 따른다.

## 네이밍
- 대상 프로젝트의 테스트 관례를 따른다(프로젝트별로 다름).
