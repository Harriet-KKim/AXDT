# test/ — 대상 프로젝트 테스트 (Test)

> 이 저장소로 개발하는 **대상 프로젝트의 테스트** 자리. AXDT 워크플로 테스트가 아니다.

## 목적
대상 프로젝트의 유닛/통합 테스트를 담는다. Tester가 사양의 **수용 기준**을 검증 가능한 테스트로 구현한다. 처음엔 비어 있다.

> **AXDT 자체 구현의 테스트는 여기가 아니다.** 그것은 `WIP/` 쪽에 둔다(D12).

## 필수 내용
- `src/`의 구현과 `docs/sot/specification/`의 수용 기준에 대응하는 테스트.
- 테스트 레이아웃·실행 방법은 대상 프로젝트 규약을 따른다.

## 네이밍
- 대상 프로젝트의 테스트 관례를 따른다(프로젝트별로 다름).
