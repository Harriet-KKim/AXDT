# ADR-0020: half-state(창-사망·컨테이너 생존)의 backend 생존 계약을 ADR-0016 보완으로 별도 확정한다

## 상태
Proposed (2026-07-23) · 소유 Phase 3

## 맥락
half-state는 tmux 창은 사라졌는데 컨테이너 프로세스는 아직 실행 중인 어중간한 상태다. 생존 검사가 `is_alive() = 컨테이너 실행 AND 창 존재`로 정의돼 있어(`infra/backend.py`의 `is_alive`), 창만 사라져도 죽음으로 읽힌다.

- 위험: 창-사망만 보고 재기동하면 살아 있는 컨테이너를 죽인다. 재기동 경로가 `reuse_workspace`에서 보상 정리(`teardown(force=True)`)를 하지 않는 규칙(Phase 2 스펙 §7.2)과 겹쳐 미커밋 작업을 잃을 수 있다.
- ADR-0016은 상태판정을 훅 상태 파일로 옮기면서 `ERROR`·`STOPPED`를 프로세스 생존(`is_alive`·`exit_code`·`last_error`)으로 판정하되, `is_alive()`를 원시연산으로 전제했다. half-state는 바로 그 `is_alive()`의 미명세 경계라 ADR-0016 범위 밖이다.
- Phase 2 설계 스펙 정합화 재검토(2026-07-22, Codex-Sol·Fable 5라운드)에서 이 긴장이 드러났다. 한 재기동 관문(§7.3)만 half-state로 특수처리한 편집이 문서 분열을 만들어 되돌렸고, 스펙은 모든 관문이 `is_alive()`를 쓰는 균일 상태로 유지했다. 통일 방식은 별도 결정으로 남겼다.

## 결정
half-state를 Phase 2 정합화에 섞지 않고, ADR-0016을 보완하는 별도 결정으로 다룬다.

- backend 생존 계약을 세 갈래로 확정한다. 권위 있는 판정은 backend `post_mortem`이 낸다.
  - 깨끗한 죽음(컨테이너 종료) → 재기동한다.
  - 실행 중 창-사망(half-state) → 재기동하지 않고 블로커로 올린다. 살아 있는 컨테이너를 죽이지 않는다(Phase 2 §2.4 하한선).
  - 완전 소멸(컨테이너·창 모두 없음) → 정리됨으로 처리한다.
- 창-사망 세션의 재부착 가능 여부(창을 다시 붙일 수 있나)를 명시한다.
- 소유·구현은 재기동 substrate가 있는 Phase 3(`TmuxDockerBackend`·`post_mortem`·`restart`), 소비는 Phase 4 `recover`다. Phase 2는 하한선만 유지한다.

구체 메커니즘(세 갈래를 가르는 판정 신호, 재부착 절차)은 실 CLI·컨테이너 거동에 의존하므로 Phase 3 구현·측정에서 확정한다. 본 ADR은 방향과 소유만 고정하며, 그래서 `status=proposed`다.

## 결과
**좋은 점**
- 재기동이 살아 있는 컨테이너를 죽이는 경로가 계약으로 닫힌다.
- half-state 판정이 backend 한 곳(`post_mortem`)에 모여, 재기동 관문마다 특수처리하지 않아도 된다.
- Phase 2 스펙이 `is_alive()` 균일 상태를 유지해 자기모순이 없다.

**대가 / 주의**
- 세 갈래 판정 신호와 재부착 절차는 Phase 3 구현·측정 전까지 잠정이다 — 본 ADR은 방향만 고정한다.
- Phase 3가 구현하기 전까지 half-state는 여전히 열린 위험이다. 현재는 문서 균일로 자기모순만 없앤 상태다.

## 검토한 대안
### 대안 A — Phase 2에서 모든 재기동 관문을 half-state 3갈래로 통일
Phase 2 스펙 §2.5·§4.5·§7.3·§11을 모두 `post_mortem` 3갈래로 바꾼다. · **기각 사유**: half-state는 ADR-0016(상태판정)과 직교인 backend 생존 계약이라 Phase 2 정합화 범위 밖이고, 한 관문만 고치면 문서가 분열한다(재검토에서 실제로 발생해 되돌림). backend 계약으로 한 곳에 모으는 편이 옳다.

### 대안 B — `is_alive()` 정의를 "컨테이너 실행"만으로 완화
창 존재 조건을 빼 창-사망을 죽음으로 안 본다. · **기각 사유**: 창이 없으면 `attach`·주입·화면 열람 경로가 성립하지 않으므로, "살아 있다"고 보고 주입하면 다른 실패가 난다. 생존과 재부착 가능성은 별개 축이라 `is_alive()` 완화로는 풀리지 않는다.
