# docs/interim/ADR/ — 아키텍처 결정 기록 (대상 프로젝트)

> 이 저장소로 개발하는 **대상 프로젝트의** 아키텍처 결정 기록(ADR).

## 목적
대상 프로젝트를 만들며 내린 비자명한 설계 결정의 맥락·결정·결과·대안을 남긴다. 나중에 "왜 이렇게 했나"를 추적·재검토할 수 있게 한다.

> **AXDT 자체 설계 ADR과 혼동 금지.** AXDT 워크플로 자체의 설계 결정(Maintainer 모델, 통신 모델 등)은 **저장소 루트 `WIP/adr/`**에 있다(D12/D13). 이 디렉터리는 AXDT로 *만들어지는 프로젝트*의 ADR 자리로, 처음엔 비어 있다.

## 필수 내용
- **번호·제목** / **상태** (제안·채택·폐기·대체)
- **맥락** — 결정이 필요한 배경
- **결정** — 무엇으로 정했나
- **결과** — 채택의 득실·영향
- **검토한 대안** — 버린 선택지와 이유

## 네이밍
- 파일명: `NNNN-<slug>.md` (4자리 0패딩, 예: `0001-database-choice.md`).
- frontmatter `id`: 대문자 `ADR-NNNN` (예: `ADR-0001`). 본문·참조도 대문자 표기.
