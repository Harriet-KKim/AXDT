# docs/sot/requirements/ — 요구사항 (Requirements)

> 대상 시스템에 **무엇이 왜** 필요한가 — 기능/비기능 요구의 권위 정의.

## 목적
시스템이 만족해야 할 목표와 제약을 확정한다. "어떻게 구현하는가"(사양)가 아니라 "무엇이 왜 필요한가"를 기술한다. 사양·테스트·수용 판정의 상위 근거다.

## 필수 내용
- **목표** — 이 요구가 달성하려는 것
- **기능 요구** — 시스템이 해야 할 동작
- **비기능 요구** — 성능·보안·가용성·운영 등 품질 속성
- **제약** — 기술·일정·환경 제약
- **범위 외** — 명시적으로 다루지 않는 것
- **수용 기준** — 요구 충족 판정 기준

## 네이밍
- 파일명: 주제 기반 kebab-case, `<topic>.md` (예: `user-accounts.md`). 번호 미사용.
- frontmatter `id`: 소문자 `req-<topic>` (예: `req-user-accounts`).
