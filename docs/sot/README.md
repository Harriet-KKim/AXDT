# docs/sot/ — Source of Truth (권위본)

> 시스템이 "참"으로 삼는 기준 문서. **변경은 사용자 게이트가 있는 PR로만** 가능하다.

## 목적
대상 프로젝트가 **무엇을·왜·어떻게 검증하며·어떤 규칙으로** 만들어지는지를 확정하는 권위 문서를 담는다. Agent는 SoT를 읽어 작업하고, SoT를 바꾸려면 PR을 올려 **사용자(Reviewer 게이트)** 승인을 받아야 한다.

## 필수 내용
- **`requirements/`** — 무엇이 왜 필요 (기능/비기능 요구)
- **`specification/`** — 시스템이 무엇을 어떻게 (동작·인터페이스·데이터)
- **`test-design/`** — 요구·사양을 어떻게 검증 (테스트 조건·커버리지·스위트·추적·절차 원칙)
- **`rule/`** — 프로젝트 규칙 (용어·네이밍·통신·표준). AXDT 베이스 규칙이 신규 프로젝트로 전파되는 자리.
