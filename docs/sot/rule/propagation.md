# 베이스 규칙은 신규 프로젝트로 상속된다

## 규칙문
> `docs/sot/rule/`의 베이스 규칙은 **AXDT 템플릿을 복제해 새 프로젝트를 시작할 때 함께 상속**된다. 상속은 **복제 시점**에 일어나며, 이후 AXDT 베이스 규칙 변경이 기존 프로젝트로 자동 역전파되지는 않는다. 신규 프로젝트는 이 규칙들을 출발 기준선으로 삼고, 프로젝트 고유 규칙은 같은 디렉터리에 **추가**한다.

## 근거
- AXDT는 프로젝트마다 **repo를 복제해 시작하는 템플릿**이다. 베이스 규칙을 repo 안에 두면 복제 시점에 자동으로 따라가, 별도 배포 메커니즘 없이 전파가 성립한다.
- 모든 프로젝트가 동일한 **통신·상태·문서 규범** 위에서 동작하도록 보장한다.

## 적용범위
- **대상**: AXDT 템플릿에 포함된 `docs/sot/rule/`의 **베이스 규칙**(프로젝트가 복제 후 추가한 고유 규칙은 그 프로젝트 소유이지 상속 대상이 아니다).
- **예외**: 프로젝트가 명시적으로 대체·폐기(`status: deprecated`/`superseded`)한 규칙. 단 rule도 SoT이므로 대체·폐기는 **사용자 게이트 PR**을 거치고(`rule-sot-change-user-gate`), 해당 프로젝트의 rule 문서에 사유를 남긴다.

## 예시
**준수 (✓)**
- 새 프로젝트 시작 = AXDT repo 복제 → progress 단일 작성자·SoT 게이트 등 베이스 규칙이 그대로 적용.

**위반 (✗)**
- 베이스 규칙을 무시하고 프로젝트마다 통신 규범을 임의로 새로 만듦. → 전파·일관성 상실.
