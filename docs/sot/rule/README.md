# docs/sot/rule/ — 규칙 (Rule)

> 프로젝트가 따르는 규칙(용어·네이밍·통신·표준). AXDT **베이스 규칙이 신규 프로젝트로 전파**되는 자리.

## 목적
프로젝트 전반에 적용되는 규범을 한곳에 모은다. 여기 담긴 AXDT 베이스 규칙은 신규 프로젝트가 저장소를 복제할 때 함께 상속된다(전파). 프로젝트 고유 규칙도 같은 형식으로 추가한다.

## 필수 내용
규칙 문서 1개당:
- **규칙문** — 한 문장으로 단정한 규칙 (blockquote)
- **근거** — 왜 이 규칙인가
- **적용범위** — 대상 / 예외
- **예시** — 준수(✓) / 위반(✗)

## 네이밍
- 파일명: 주제 기반 kebab-case, `<topic>.md` (예: `terminology.md`). 번호 미사용.
- frontmatter `id`: 소문자 `rule-<topic>` (예: `rule-terminology`). 파일명에 `rule-` 접두는 붙이지 않는다(중복 방지).
